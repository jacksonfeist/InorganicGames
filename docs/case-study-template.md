# Case study template

Build the component now so the slot is ready; fill it the day there's a real client result. Do not fabricate one to fill it early — see the honesty rules at the bottom.

## Until we have a real result

The Work page is **spec demos + methodology framing**, and that is fine and honest. A concept build presented as a concept build, with the thinking behind the hook made explicit, is credible. A concept build dressed up as a campaign is a lie a buyer will catch on the first follow-up question.

## Required fields

A credible playable case study — the convention across the trustworthy examples in the market — contains all of these:

### 1. The setup, named specifically

> Casual merge game, iOS + Android, US market, Feb–Mar 2026.

Named game, genre, platform, market, timeframe. Vagueness here reads as fabrication.

### 2. The baseline table — **before** the results

Quantify the problem first. This is the single thing that separates a credible case study from a vanity one, and almost nobody does it.

### 3. The mechanic, named

Describe what the interactive loop actually *was*:

> "Raining-coins hook → 3-tap merge loop → install CTA."

If you can't name the mechanic in one line, the build didn't have one.

### 4. Multi-metric results

**CPI + IPM + retention together. Never CTR alone** — CTR is gameable and leading with it signals amateurism (see `docs/positioning.md`).

Ideally show month-by-month progression rather than a single before/after jump. A progression looks like a real campaign; a single leap looks like a cherry-picked screenshot.

### 5. Disclosed test method and sample size

> "≥100 conversions per creative over 7 days."

**Almost no vendor case study discloses this.** Doing it makes us visibly more rigorous than the field, and it's free — we just have to actually run a clean test.

### 6. A named client quote — optional

A supplement to hard numbers, **never a substitute** for them. A glowing quote attached to no metrics reads as an admission that the metrics weren't good.

## Honesty rules

- Never present a spec build as a paid campaign.
- Never invent or estimate a number and present it as measured.
- If a result was mixed, say so. A case study that admits "CPI improved, retention didn't move" is enormously more believable than one where everything went up, and the buyer has seen a hundred of the latter.
- Our legitimacy is the entire positioning. It is worth more than any single number we could make up.
